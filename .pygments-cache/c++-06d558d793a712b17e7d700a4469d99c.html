<code>
In a way, references work similarly to pointers with more limitations and are `const` in nature. The existence of references, however, is not fully redundant - operations involving references are generally safer than those involving pointers, and are usually used unless the situation demands methodalities available only with pointers. References are also generally easier to use, as there's no need to keep track of dangling pointers or inaccessible memory. While I won't be going through the in-depth uses of references, I've written one instance that showcases their functionality and definition well.
</code>c++
void swap_val (int foo, int bar) {
    int qux = foo;
    foo = bar;
    bar = qux;
}
<code>
When the above method `swap_val (a, b)` is called in an external method, the parameters `foo` and `bar` are merely copies of the values stored in `a` and `b`. As a result, calling this method will not result in swapping the actual values of `a` and `b` outside the method. When this behaviour happens in a method, it's said that its parameters **pass by value**. In the alternative method below,
</code>c++
void swap_ref (int &amp;foo, int &amp;bar) {
    int qux = foo;
    foo = bar;
    bar = qux;
}
<code>
Notice that references are passed instead of actual `int` parameters. When `swap_ref (a, b)` is called, what's passed to the method are *references to a and b*, and not just copies of their values. When this behavior occurs, a method's parameters are said to **pass by value**. Now, when the method calls operations on references of `a` and `b`, they are actually called on the variables `a` and `b` passed as parameters as well. As a result,
</code>c++
int foo = 10;
int bar = 20;
swap_val (foo, bar);
std::cout &lt;&lt; foo &lt;&lt; std::endl; // Output: 10
std::cout &lt;&lt; bar &lt;&lt; std::endl; // Output: 20</p>

swap_val (foo, bar);
std::cout &lt;&lt; foo &lt;&lt; std::endl; // Output: 20
